# memory-and-context 规格说明

## 目的

定义 K-Chat 如何保存对话上下文、如何压缩较早消息，以及如何持久化长期用户画像数据。

## 要求

### Requirement: 保留最近对话窗口

系统 SHALL 保留一个有界的最近对话窗口，以维持连续上下文。

#### Scenario: 最近消息数量超过配置窗口

- **WHEN** 新消息加入后超出配置的最近历史上限
- **THEN** 只有最近的一部分消息会保留在活跃的短期窗口中

### Requirement: 可配置的摘要压缩

当对话长度超过配置的摘要阈值后，系统 SHALL 支持把更早的消息压缩成摘要。

#### Scenario: 对话历史超过摘要阈值

- **WHEN** 摘要记忆判断当前对话规模已经适合压缩
- **THEN** 系统会总结较早的消息，并保留最近的一部分原始消息

### Requirement: 持久化用户画像记忆

系统 SHALL 在进程内存之外持久化长期用户画像数据。

#### Scenario: 一次新的聊天请求开始

- **WHEN** 服务端准备回答上下文
- **THEN** 它会加载已持久化的用户画像，并把它作为模型上下文的一部分

### Requirement: 去重的画像增量合并

系统 SHALL 在合并兴趣、事实和偏好时避免保存重复语义的条目。

#### Scenario: 长期记忆接收到重复用户事实

- **WHEN** 画像更新中包含重叠的兴趣、事实或偏好
- **THEN** 持久化后的画像会保存规范化后的合并结果

### Requirement: 面向 Prompt 的可读上下文渲染

系统 SHALL 将长期记忆渲染成适合拼接进 Prompt 的可读上下文字符串。

#### Scenario: 画像字段存在可用值

- **WHEN** 画像中包含姓名、职业、兴趣、偏好或已知事实
- **THEN** 系统会把它们格式化成可读的上下文块，而不是直接输出原始 JSON
